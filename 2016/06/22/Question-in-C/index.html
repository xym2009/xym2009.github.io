C++编译问题

2016-06-22 • 分类: -study • 标签: -C++ -error

　　总结一下在C++编译过程中遇到过的一些问题，环境为VS2010.

  1. error LNK2019:无法解析的外部符号
    在 项目–属性–链接器–常规–附加库目录，填写附加依赖项所在目录，用分号间隔多项。同时在 项目–属性–链接器–输入–附加依赖项，填写附加依赖项的名字.lib，用分号间隔多项。

  2. fatal error C1902:程序数据库管理器不匹配；请检查安装
    通常错误是指 mspdbsrv.exe 或 mspdbcore.dll 丢失 或者其版本与 mspdb80.dll的版本不同，确定这三个文件的相同版本安装在系统即可，目录一般为\Program Files\Microsoft Visual Studio 10\VC\bin.

  3. error LNK2005:已经在*.obj中定义的问题
    全局变量的重复定义，在头文件中加入 #pragma once即可。

  4. error LNK2005:已经在libcmtd.lib中定义
    如果有一堆的重定义错误发生在同一个lib中，而且跟它冲突的也是同一个lib，那么这两个的功能是一样的，2选1即可。在 项目–配置–链接器–输入–忽略特定的库：libcmtd.lib;在 项目–配置–常规–MFC的使用：在共享DLL中使用MFC。

  5. error LNK1123:无法解析的外部符号
    在 项目–属性–配置–清单工具–输入和输出–嵌入清单 是改成否；在 项目–属性–配置–链接器–清单文件–嵌入清单 是改成否。

  6. C1189：#error Building MFC Application with /MD[d]
    在 项目–配置–常规–MFC的使用：在共享DLL中使用MFC。

  7. DllMain@12已经在LIBCMTD.lib(dllmain.obj)中定义
    关于动态链接库编译顺序的问题，因为.cpp文件是按照字母顺序排序的，编译时遵循这个顺序的反序，则将需要第一个编译的文件的文件名改成’Zxxx.cpp’即可

  8. 预编译头文件stdafx.cpp无法打开的问题
    将stdafx.cpp设置成创建预编译文件，其他的cpp设置成使用预编译文件。

  9. 程序数据库管理器不匹配；请检查安装
    mspdb100.dll存在在多个路径中，删除以下路径中的/Program Files/Microsoft Visual Studio 10/VC/bin；C:/Windows/System32中的相应文件即可。

  10. 不能从const char*转换为LPCWSTR的问题
    在 项目–属性–配置–常规–项目默认值–字符集，将使用Unicode字符集改成未设置即可。
